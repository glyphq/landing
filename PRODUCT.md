# Product

## Register

brand

## Platform

web

## Users

Primary audiences are Qubic application developers and people evaluating or using maintained Qubic software. They need accurate product status, direct integration guidance, safe download paths, and clear maintenance evidence.

## Product Purpose

The site establishes Glyph as an independent software organization, explains how its products fit together, and directs visitors to verified releases, packages, repositories, documentation, and contribution paths.

## Positioning

Independent software suite built for Qubic.

## Conversion & proof

- Primary CTA: explore the ecosystem or start building, depending on visitor intent.
- Secondary CTA: get Glyph Wallet.
- Remembered line: Build on Qubic without rebuilding the basics.
- Proof: public Glyph repositories, Wallet releases, Connect npm package, published security policy, and source-controlled implementation details.

## Brand Personality

Precise, calm, and infrastructural. The site should communicate maintained competence rather than market excitement.

## Anti-references

No crypto spectacle, mesh gradients, cubes, particles, coins, neon, glass panels, fabricated dashboards, speculative copy, or generic SaaS card repetition.

## Design Principles

1. Make complexity legible.
2. Reveal structure instead of creating decoration.
3. Use proof instead of adjectives.
4. Distinguish current software from future direction.
5. Keep local control and system boundaries visible.

## Accessibility & Inclusion

Target WCAG 2.2 AA with complete keyboard access, reduced-motion support, sufficient contrast, semantic diagrams, and responsive content from 320px upward.
